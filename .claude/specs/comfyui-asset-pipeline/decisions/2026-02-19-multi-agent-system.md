# Decision: 멀티에이전트 팀 시스템 도입

- **날짜**: 2026-02-19
- **상태**: 승인됨
- **영향**: 전체 프로젝트

## Context
FlowSpace는 flow_metaverse 리팩토링 프로젝트로, 여러 도메인(게임엔진, 에셋, 통신, 프론트엔드, 백엔드)에 걸친 대규모 작업이 필요함.

## Decision
5개 도메인 에이전트 + 오케스트레이터 체제를 도입하고, FlowHR의 Contract Governance 패턴을 적용하여 에이전트 간 인터페이스를 관리한다.

## Alternatives Considered
1. **단일 에이전트**: 모든 도메인을 하나의 에이전트가 처리 → 컨텍스트 윈도우 한계
2. **도메인별 독립 작업**: contract 없이 각자 작업 → 인터페이스 불일치 위험

## Consequences
- **장점**: 도메인 전문성, 병렬 작업, 명확한 경계
- **단점**: 인프라 오버헤드, 에이전트 간 조율 비용
- **리스크**: 백그라운드 에이전트 권한 이슈 (실제 발생 → 오케스트레이터가 직접 실행으로 해결)
